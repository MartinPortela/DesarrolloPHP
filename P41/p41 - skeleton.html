1. Audio y vídeo en HTML5

Qué es / para qué sirve

Sirve para insertar audios y vídeos en nuestra página web para permitir interactividad.

Casos de uso típicos

  • Reproductores de música
  • Para mostrar el adelanto de una película en nuestra página de cine
  • Notificaciones sonoras
  • Podcasts y audiolibros

Ventajas

  • No requiere plugins externos
  • Compatibilidad multiplataforma
  • Integración sencilla
  • Accesibilidad
  • Soporte responsive

Limitaciones

  • Dependencia del ancho de banda
  • Restricciones de autoplay
  • Se queda corto para el streaming avanzado
  • Protección de contenido limitada

Ejemplos reales (webs conocidas)

Vimeo, Soundcloud y Discord

Conclusión personal

Usaría HTML5 para crear páginas web modernas e interactivas porque permite integrar audio y vídeo sin plugins externos, también resulta útil por su utilidad para hacerlo compatible con móviles y distintos navegadores añadido a su facilidad de implementación. Es ideal para aplicaciones web dinámicas gracias a su integración con JavaScript y CSS, finalmente, facilita el desarrollo web porque es un estándar moderno, rápido y ampliamente soportado.

2. Canvas

Qué es / para qué sirve

Define canvas y qué tipo de gráficos/interacciones permite.

El elemento canvas de HTML5 es un área de dibujo que permite crear gráficos y animaciones dinámicamente mediante JavaScript. Funciona como una “superficie en blanco” donde el programador puede dibujar y actualizar contenido en tiempo real.

Permite crear gráficos 2D y 3D, animaciones, videojuegos, visualización de datos, editores de imágenes y simulaciones interactivas.

Casos de uso típicos

  • Videojuegos web
  • Gráficos y visualización de datos
  • Animaciones y efectos visuales
  • Mapas y geolocalización interactiva

Ventajas

  • Alto rendimiento
  • Gran flexibilidad
  • Integración con JavaScript
  • Compatible con navegadores modernos
  • Experiencias visuales interactivas

Limitaciones

  • Accesibilidad limitada
  • Consumo de recursos
  • Requiere más programación
  • Mantenimiento más complejo
  • No ideal para interfaces tradicionales

Ejemplos reales (webs conocidas)

Agar.io, Youtube y Spotify

Conclusión personal

Elegiría Canvas frente a SVG o HTML cuando necesito trabajar con gráficos muy dinámicos o en tiempo real, como videojuegos, simulaciones o visualizaciones con muchos elementos que cambian constantemente. Esto es porque Canvas dibuja píxeles directamente en una superficie, lo que le da muy buen rendimiento cuando hay muchas actualizaciones por segundo.

3. SVG

Qué es / para qué sirve

SVG (Scalable Vector Graphics) es un formato gráfico vectorial basado en XML que permite crear imágenes escalables sin perder calidad. Es muy útil para iconos, logos, ilustraciones, gráficos y animaciones porque puede adaptarse a cualquier resolución o tamaño de pantalla.

Casos de uso típicos

  • Logotipos e iconos
  • Gráficos y diagramas
  • Mapas interactivos
  • Animaciones vectoriales
  • Interfaces responsive

Ventajas

  • Escalable sin pérdida de calidad
  • Poco peso en gráficos simples
  • Accesible y compatible con SEO
  • Permite animaciones
  • Fácil integración con CSS y JavaScript

Limitaciones

  • No ideal para gráficos muy complejos
  • Puede perder rendimiento con muchos elementos
  • Más difícil para imágenes realistas
  • Animaciones complejas requieren más trabajo

Ejemplos reales (webs conocidas)

Wikipedia, GitHub y Google Maps

Conclusión personal

Usaría SVG cuando necesite gráficos escalables y ligeros como iconos, logotipos o ilustraciones responsive. También lo elegiría para gráficos interactivos o animaciones simples porque mantiene buena calidad en cualquier resolución y se integra fácilmente con CSS y JavaScript.

4. CSS Animations / Transitions

Qué es / para qué sirve

Las transiciones y animaciones CSS permiten crear efectos visuales e interacciones dinámicas sin necesidad de JavaScript.

Las transiciones cambian suavemente una propiedad entre dos estados, mientras que las animaciones permiten movimientos y efectos más complejos mediante keyframes.

Casos de uso típicos

  • Efectos hover en botones
  • Menús desplegables
  • Animaciones de carga
  • Transiciones entre elementos
  • Interfaces modernas e interactivas

Ventajas

  • Fáciles de implementar
  • Buen rendimiento
  • No requieren JavaScript
  • Código más limpio
  • Compatibles con navegadores modernos

Limitaciones

  • Menos flexibles que JavaScript
  • Animaciones complejas son difíciles
  • Compatibilidad parcial en navegadores antiguos
  • Limitadas para lógica avanzada

Ejemplos reales (webs conocidas)

Apple, Netflix y Airbnb

Conclusión personal

Usaría CSS Animations y Transitions para efectos visuales sencillos y animaciones ligeras porque ofrecen buen rendimiento y un código más limpio. Las elegiría frente a JavaScript cuando la animación no necesite lógica compleja ni interacción avanzada.

5. JavaScript para interactividad

Qué es / para qué sirve

JavaScript permite añadir interactividad a las páginas web respondiendo a eventos del usuario y modificando el DOM dinámicamente.

Gracias a JavaScript se pueden crear aplicaciones web dinámicas, validar formularios, mostrar contenido en tiempo real y controlar elementos multimedia.

Casos de uso típicos

  • Validación de formularios
  • Menús interactivos
  • Aplicaciones SPA
  • Videojuegos web
  • Actualización dinámica de contenido

Ventajas

  • Alta interactividad
  • Compatible con todos los navegadores modernos
  • Gran flexibilidad
  • Permite modificar el DOM en tiempo real
  • Compatible con APIs modernas

Limitaciones

  • Puede afectar al rendimiento
  • Mayor complejidad del código
  • Problemas de seguridad si se usa mal
  • Dependencia del navegador del usuario

Ejemplos reales (webs conocidas)

Facebook, YouTube y Gmail

Conclusión personal

Usaría JavaScript cuando necesite interactividad avanzada o modificar contenido dinámicamente. Sin embargo, evitaría usarlo para tareas simples que puedan resolverse con HTML o CSS para mantener un mejor rendimiento y un código más sencillo.

6. APIs relacionadas

Incluye mínimo 2 APIs

API 1: WebRTC

Qué hace: Permite comunicación en tiempo real de audio, vídeo y datos entre navegadores.

Casos de uso: Videollamadas, chats de voz, streaming y compartir pantalla.

Ejemplos reales: Google Meet, Discord y Zoom.

Conclusión personal: Es una API muy útil para aplicaciones de comunicación en tiempo real porque permite crear videollamadas directamente desde el navegador sin instalar programas externos.

API 2: Web Audio API

Qué hace: Permite procesar y manipular audio en tiempo real mediante JavaScript.

Casos de uso: Visualizadores de audio, efectos sonoros, sintetizadores y videojuegos.

Ejemplos reales: Spotify, SoundCloud y juegos web interactivos.

Conclusión personal: Usaría esta API para proyectos multimedia interactivos porque ofrece mucho control sobre el audio y permite crear experiencias dinámicas.

API 3: MediaRecorder API

Qué hace: Permite grabar audio y vídeo directamente desde el navegador.

Casos de uso: Grabación de videollamadas, notas de voz y captura multimedia.

Ejemplos reales: Loom y aplicaciones de grabación online.

Conclusión personal: Me parece útil para aplicaciones multimedia modernas porque facilita la grabación de contenido sin instalar software adicional.
